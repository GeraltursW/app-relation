# Node Editing API

## Data contract

Graph nodes returned by `GET /queryAppGraph/{app_name}` include:

- `id`: response-local numeric node identifier used by the frontend graph layout
- `page_id`: stable hash identifier used by persistence APIs
- `images`: screenshot filename array; the first item is the primary graph image
- `ai_inference`: object containing AI inference fields
- `action`: reviewed page-action object with four fixed arrays

Do not use `id` as `page_id`.

## Update node

`POST /update_node`

Content type: `multipart/form-data`.

Fields:

| Field | Type | Required | Meaning |
| --- | --- | --- | --- |
| `page_id` | string | yes | Stable page hash |
| `page_title` | string | yes | Page display title |
| `page_text` | string | yes | AI page description |
| `page_url` | string | yes | Big-data or application URL |
| `widget_description` | string | yes | Parent control used to enter this page |
| `ai_inference` | JSON string | no | Object such as `{"label":"...","reason":"..."}` |
| `action` | JSON string | no | Complete reviewed action object; saved as one unit |
| `ai_recursive` | boolean | no | Whether the page was generated by AI exploration |
| `keep_images` | JSON string | no | Existing filenames to retain |
| `new_images` | repeated file | no | Newly uploaded screenshots |

The final image list is:

```text
images = keep_images + stored filenames generated from new_images
```

Response:

```json
{
  "statue": "success",
  "node": {
    "page_id": "stable-hash",
    "page_title": "Page title",
    "page_text": "Description",
    "page_url": "/page/url",
    "widget_description": "Parent widget",
    "images": ["stored-file.png"],
    "ai_inference": {
      "label": "Inference label",
      "reason": "Inference reason"
    }
  }
}
```

Uploaded images are read through `GET /image/{image_name}`.

## Action contract

`page_actions` is replaced by `action`:

```json
{
  "popupAction": [],
  "stateAction": [],
  "externalAction": [],
  "pageNaviAction": []
}
```

Each array contains objects. Common fields are `id`, `label`, `action_type`, and
`description`. Category-specific fields include:

- `popupAction`: `popup_name`
- `stateAction`: `state_key`, `state_value`
- `externalAction`: `target`
- `pageNaviAction`: `target_page_id`, `target_page_title`

When a page has never been manually reviewed, `queryAppGraph` derives
`pageNaviAction` from its outgoing edges. Once `/update_node` receives an
`action` object, that complete object becomes authoritative. Empty arrays are
therefore preserved as an intentional human-review result.
